# ai-fde

- Transformed career artifacts from document-centric records into structured project knowledge, preserving trade-offs, architectural decisions, and outcomes as reusable source material for resume, interview, and portfolio generation.

- Developed a layered knowledge model separating memory, context, policy, outcomes, and decisions, enabling the same project knowledge to support multiple representations with different levels of detail.

- Established shared information architecture and writing standards to make project knowledge consistently reusable across projects, roles, and career artifacts while reducing repeated analysis and maintenance effort.

- Designed a feedback-driven evolution model connecting outcomes, reflection, and knowledge updates to explore how career systems can continuously improve interview narratives, project positioning, and skill-development priorities.

## Evidence Mapping

Bullet 1

    Claim:
        Resume, interview, and portfolio documents were transformed into reusable project knowledge.

    Sources:
        - Problem Space: Project files exist; project knowledge does not.
        - Problem Space: Architecture decisions, trade-offs, discoveries, and capabilities remain implicit.
        - Current Capability: Knowledge Creation defines reusable understanding as problems, constraints, trade-offs, decisions, and outcomes.
        - Story: Documents Were Not The Real Problem identifies that facts existed, but reusable understanding did not.

Bullet 2

    Claim:
        A layered knowledge model separates storage, selection, governance, outcomes, and future recommendations.

    Sources:
        - Architecture: Memory Layer stores accumulated project knowledge.
        - Architecture: Context Layer selects relevant context for resume, interview, and portfolio goals.
        - Architecture: Policy Layer defines information architecture, writing format, and role profiles.
        - Architecture: Outcome Layer captures application and interview outcomes.
        - Architecture: Decision Layer recommends future improvements.
        - Current Capability: Knowledge Representation supports Knowledge → Resume, Knowledge → Interview, and Knowledge → Portfolio.

Bullet 3

    Claim:
        Shared structure and standards make knowledge reusable across career artifacts while reducing repeated work.

    Sources:
        - Current Capability: Knowledge Structure keeps knowledge consistent through problem space, capabilities, architecture, stories, information architecture, and writing standards.
        - Current Capability: Knowledge Structure benefits include consistent outputs, lower maintenance cost, and easier knowledge reuse.
        - Current Capability: Knowledge Creation benefits include reducing repeated analysis and preserving architectural reasoning.
        - Problem Space: One representation creates one granularity and many rewrites.

Bullet 4

    Claim:
        The system direction adds feedback loops so real-world outcomes can improve future narratives, positioning, and skill priorities.

    Sources:
        - Future Problem: Feedback from applications, interviews, and skill gaps is not captured.
        - Future Capability: Outcome Capture supports Application → Outcome, Interview → Outcome, and Skill Assessment → Outcome.
        - Future Capability: Action Guidance combines knowledge, feedback, and goals to recommend actions.
        - Architecture: Dynamic Flow moves from Experience → Knowledge → Context Selection → Outcomes → Reflection → New Knowledge.
        - Architecture: Decision Layer examples include which skills deserve investment, which experiences deserve emphasis, which narratives deserve refinement, and which content should be removed.
