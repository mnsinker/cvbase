# Summary

This glossary defines the tagging system used throughout the CV Base.
The tagging system consists of 3 layers.

Career Spine

    Answers: What was the primary object being modeled? (中文: 我当时主要在建模什么?)


Advanced Themes

    Answers: What capability was being improved or explored? (中文: 优化风向)


Project Tags

    Answers: What platform, domain, system, or technology does this project involve?


# Career Spine

    Purpose:
        Define career growth trajectory
        Support career narrative generation
        Connect projects into a coherent story
    Applied at: Story level
    Allowed Values: Only use values defined below.



## information

Core question: How should information be presented and understood?

Examples:

    UI Design
    UX Design
    Information Architecture
    Conceptual Model

---

## workflow

Core question: How should work be organized and executed?

Examples:

    Business Process Design
    Studio Operations Workflow
    PMS Workflow
    Approval Flow


## data

Core question:

    What business facts should be captured?

Examples:

    Financial Summary Table
    Single Customer View
    Reporting Logic
    Semantic Alignment


## systems

Core question: 
    
    How do multiple workflows and data interact?

Examples:

    OMS
    CRM
    CDP
    SAP
    Enterprise Platforms
    Cross-System Integration


## decisions

Core question: 
    
    How should systems determine what happens next?

Examples:

    Policy Engine
    Decision Engine
    Recommendation Systems
    Adaptive Learning Systems
    Planning Systems
    



# Advanced Themes

    Purpose:
        Highlight higher-order interests
        Surface areas of specialization
        Support interview and portfolio generation
    Applied at: Story level
    Allowed Values: Only use values defined below.


## automation

Core question:

    How can execution become less manual?

Examples:

    Airtable Automation
    Flowable
    Workflow Automation
    Agent Execution



## governance

Core question:

    How can systems remain reliable?

Examples:

    EQ Protocol
    Approval Process
    Policy Layer
    Operating Model



## ai-systems

Core question:

    How can AI participate in system execution and decision-making?

Examples:

    Question Forge
    Decision Engine
    Agent Runtime
    Context Architecture



# Project Tags

Purpose:

    Purpose: 
        Support filtering and retrieval
        Support recruiter keyword matching
        Support domain-specific portfolio views
        They are not part of the core career narrative.
    Applied at: Project level.
    Allowed Values: No strict values are enforced. Use concise tags that describe:
        - domains: education, retail, healthcare, operations, marketing
        - systems / tools: oms, crm, cdp, sap, airtable, flowable, wechat-mini-program
        - technologies: llm, vlm, agent, ontology, policy-engine
